python值的传递

Posted on 2019-10-10 | In python

How does objective be passed in Python?

这几天学习Python和帮同学解决问题中遇到了一个神奇的小问题。

如果指定一个list为a，运行b=a，对b做更改，a的值也会同样更改。这引起了我深入了解python中值的传递的深层原理。

python中一切皆对象，这句话刚开始学习的时候并没有任何感觉。

但随着深入了解编程，python的确大不同于C或者java。所以这里很难说一个整数、浮点数或者一个list是value或者reference。所以这里用objective，记录我学到的Python是如何传递和引用对象的。

事实上，python中有两种objective: mutable & immutable

  1. mutable包含list, set, dict

  2. immutable包含bool, int, float, tuple, str, frozenset

先看几个例子：

immutable

看一下int：

可以看到x是一个int, 用 “ = ” 符号对y进行复制后，x、y有着相同内存地址。在对y自加1后，y有了不同的内存地址。数字1、2的地址与之分别对应。

也就是在python中，immutable数据存储在内存地址中，对应的变量指向该内存地址。如果对Immutable数据进行了改变，会生成一个新的数值存储在新的内存地址。

mutable

以list为例子：

我们可以看到，用等号对y赋值后，x,y具有相同的内存地址。对x进行操作并不改变内存地址。与此同时，指向同一内存地址的y的值同时改变。也就mutable变量的内存地址是不变的。

但我们也可以发现，mutable内部如果有immutable的类型如x[1]是int，在对x[1]进行改变后。X[1]的内存地址改变了。也就是现在的X[1]不再是之前的X[1]

如何直接复制到一个新的内存地址

使用copy可以复制到一个新的内存地址。

同时要注意，对应immutable变量，如果令 x =1, y = 1, x与y会有同样内存地址。而对应mutable变量则不同，会产生不同的内存地址。

id() 与 is()

id() 用来返回内存地址

is() 用来判断内存地址是否相同

对于immutable的内存地址，不同解释器似乎有不同设置。本人水平有限无法进行讨论。
